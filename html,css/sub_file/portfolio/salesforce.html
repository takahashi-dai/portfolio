①企画

顧客データや商談データをExcelで管理していたが、社内ルールとして決まっておらず、データが散乱。
なおかつデータが膨大になってきておりExcelでは不便を感じたため、SalesForceの導入を考え始め企画した。
無料のBIツールからの導入も考えたが、自由度が高く営業戦略を立てられるものを見つけられなかったため断念。

②ヒヤリング

思い返すと一番の失敗がこのヒヤリング段階。
今まで企画提案をした経験が無かった上に、SalesForceの知識もあまり無かった為、円滑に進める事が出来なかった。
出来ること、出来ない事を把握していない状態でのヒヤリングは地獄を作り出し、開発側も使用側も混乱を招いた。

③勉強

流石に知識が無さ過ぎたため毎日勉強の日々。
しかしSalesForceにおいての勉強は過酷なものがあり、会社の業務内容によって内容が異なるものであり、
自分が作るものに適した情報を見つけるのに一苦労。
結果、SalesForceコミュニティや、公式のチュートリアルを片っ端から調べ、テスト環境で一度作り検証。
そのような事を何度も何度も繰り返し、基本的な昨日は大体把握することが出来た上で、再度会議へ。
基本的な機能から、作れるもの、作れないものの資料を作成し、毎週の会議で発表。
ウォーターフォールのような開発工程を想像していたが、無理を悟りアジャイルでの開発を決行。

④開発

開発側が自分ひとりということもあり、自由に作れる半面、分らないこともたくさんあった。
PDCAサイクルの要領でひたすらに作業をし、約1年間勉強と作業に没頭した。
作りこみすぎても後のメンテナンスに支障をきたす為、Apexや、VisualForceの技術的なところは控えめにし、
使用者側の事も考慮し、あくまでデータ抽出、検索に適す形に仕上げ、データ管理はExcelを使用するようにした。
各部署に適した形のページを作り上げ、それぞれが使いたいデータ、見たいデータを明白にし、
部署間での円滑なやりとりを実現することが出来た。
中には必要性を感じず、疑問を抱く人もいた為、分かりやすい説明も必要とされた。

⑤実装

営業で使われるデータは速さを重視されたため、salesforceの実装により、瞬時にデータを割り出すことが可能となった。
不動産としても物件情報や、入居者情報といった細かいデータも綺麗にまとめ上げることが出来た。
今まで物件管理をやっていた人や、顧客データを入力してる人、商談データを基に営業戦略を立てる人などなど
結果としては良い業務改善となり実装して良かったと感じた。
このような経験をさせて貰ったことはすごく貴重な体験であり、自分の成長をとても感じた良い機会となった。
